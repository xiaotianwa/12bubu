# 一二布布 PC 桌面宠物设计稿

## 目标定位

做一个多功能、治愈、低打扰的 PC 桌面宠物。它不只是“会动的贴纸”，而是一个以一二布布为人格入口的小型桌面助手：平时像住在桌面角落的小伙伴，会发呆、打盹、被拖动、偶尔互动；需要时可以展开提醒、便签、计时、快捷启动、天气、日程等功能。

产品气质应该是“功能藏在软萌下面”。默认状态轻、安静、陪伴感强；功能层通过气泡、抽屉、小卡片出现，避免把布布做成一个笨重控制台。

## IP 视觉关键词

- 软萌圆润：头身比例偏幼态，大圆头，小短手脚。
- 深棕描边：线条较粗，圆角笔触，避免尖锐边。
- 低饱和奶油色：白色布布、浅棕伙伴、粉色脸颊、奶黄小道具。
- 表情克制：点眼、小嘴、脸颊传情，不做夸张拟人表情。
- 情绪轻轻的：开心、困、害羞、被戳中都要“软”，不要吵闹。

## 角色拆分

### 白色布布

主桌宠，常驻桌面。

- 白色圆身体，深巧克力色耳朵/脚/小领结。
- 粉色脸颊，小黑豆眼。
- 适合承担 idle、walk、drag、sleep、feed、tap 等大多数动作。

### 棕色伙伴

辅助陪伴角色，作为彩蛋或双宠模式出现。

- 浅棕圆身体，黄奶油脸颊。
- 动作频率低于白色布布，偶尔趴睡、冒头、贴贴。
- 可以做成“召唤伙伴”功能，避免桌面同时太拥挤。

## 交互设计

### 默认行为

- 10-20 秒随机切换一次小动作：眨眼、晃身体、看鼠标。
- 30-90 秒无操作进入半睡状态。
- 长时间无操作进入睡眠，出现小小的 Z 标记。

### 功能入口层

推荐采用“三层交互”：

1. 直接点布布：情绪反馈，不打断用户。
2. 长按 / 右键布布：展开快捷功能轮盘。
3. 点击功能卡片：打开对应的小浮窗。

这样布布本体仍然是宠物，功能不会压过 IP。

### 鼠标互动

- 单击：开心抖一下或挥手。
- 双击：弹出小菜单。
- 拖动：身体轻微压扁，松手后弹回。
- 鼠标靠近：看向鼠标或小幅后退。

### 右键菜单

- 置顶 / 取消置顶
- 透明度
- 音效开关
- 召唤棕色伙伴
- 更换动作主题
- 功能设置
- 退出

## 多功能模块规划

### P0：核心桌宠能力

- 透明置顶窗口。
- 拖拽移动。
- 动作状态机。
- 点击、双击、右键反馈。
- 位置、透明度、开机启动配置。

### P1：高频实用功能

这些功能适合第一版做，因为它们和桌宠陪伴感天然匹配。

| 功能 | 交互方式 | 布布表现 |
| --- | --- | --- |
| 待办 / 便签 | 小便签卡片 | 布布抱着小纸条 |
| 番茄钟 / 专注计时 | 计时小浮窗 | 布布认真坐下，结束时跳一下 |
| 喝水 / 休息提醒 | 气泡提醒 | 布布举奶瓶或轻轻敲屏 |
| 快捷启动 | 功能轮盘 | 布布拿出小工具箱 |
| 天气 | 小天气卡片 | 晴天戴小花，雨天撑小伞 |
| 音量 / 音乐控制 | 迷你控制条 | 布布戴耳机或跟着晃 |

### P2：增强功能

- 日程提醒。
- 剪贴板历史。
- 桌面文件整理入口。
- AI 聊天 / 灵感便签。
- 截图小工具。
- 屏幕边缘吸附。
- 多皮肤 / 多动作包。

### P3：长期玩法

- 好感度 / 成长值。
- 每日心情。
- 成就徽章。
- 棕色伙伴解锁。
- 节日限定动作。
- 用户自定义动作包。

## 功能 UI 形态

### 快捷功能轮盘

围绕布布弹出 4-6 个圆形按钮，适合：

- 便签
- 计时
- 提醒
- 天气
- 快捷启动
- 设置

视觉上像小泡泡，不要用硬质系统按钮。

### 小浮窗卡片

适合承载具体内容，例如待办、番茄钟、天气。

- 圆角 18-24px。
- 奶油白 / 淡粉 / 淡绿背景。
- 深棕文字。
- 少量阴影，但不要真实厚重阴影。
- 布布可以趴在卡片边缘，作为情绪装饰。

### 提醒气泡

适合短句提醒，例如“该喝水啦”“休息一下眼睛”。

- 气泡最多 1-2 行。
- 默认 5-8 秒后收起。
- 不抢焦点。
- 用户全屏或游戏时自动静默。

## 技术建议

如果目标是快速做出 PC 原型，推荐：

1. Electron：适合跨平台、透明无边框窗口、托盘菜单、前端动画调试快。
2. Python + PySide6：适合 Windows 小工具，打包相对简单，桌面宠物窗口控制直接。
3. Godot：适合动作状态机和动画资源管理，但桌面透明窗口需要额外处理。

我个人会优先选 Electron 或 PySide6。若只做 Windows，PySide6 的体积和行为更像原生小宠物；若后续要做功能面板、插件市场、AI 对话、账号同步，Electron 更顺。

### 多功能桌宠技术倾向

如果你想做“多功能”，我更偏向 Electron + React：

- 透明无边框桌宠窗口。
- 功能浮窗、设置页、卡片 UI 更好做。
- 后续接 AI、天气 API、日程、插件系统更自然。
- 可用前端动画管理布布动作。

如果你更在意 Windows 原生轻量，则选 PySide6：

- 体积和资源占用更可控。
- 托盘、窗口置顶、鼠标穿透等桌面能力直接。
- UI 精致度和复杂功能扩展会比 Electron 多费些手工。

## MVP 功能范围

- 透明置顶桌宠窗口。
- 一二布布 idle / walk / sleep / drag 四组动作。
- 鼠标拖拽移动。
- 快捷功能轮盘。
- 便签。
- 喝水 / 休息提醒。
- 番茄钟。
- 右键菜单和系统托盘。
- 简单随机状态机。
- 本地配置保存位置和开关。

## 风格禁区

- 不要过度 3D 化。
- 不要复杂渐变和真实毛发。
- 不要写实阴影；桌宠需要像贴纸一样干净。
- 不要加入过多 UI 图标，避免破坏 IP 纯净感。
- 不要让表情过于夸张或“表情包化”。
